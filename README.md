# kotlin-lotto-precourse

로또 구입 금액에 해당하는 만큼 로또를 발행하고, 당첨 번호와 비교해 당첨 내역과 수익률을 출력하는 프로그램이다.

## 기능 목록

### 도메인
- 로또 번호: 1 ~ 45 사이의 정수
- 로또: 중복되지 않는 6개의 번호로 구성
- 로또 발행기: 1 ~ 45 범위에서 중복되지 않는 6개의 숫자를 무작위로 추출
- 구입 금액에 해당하는 수량(금액 / 1,000)만큼 로또를 발행
- 당첨 번호: 중복되지 않는 6개의 번호와 보너스 번호 1개로 구성
- 당첨 등수 판정
  - 1등: 6개 번호 일치 / 2,000,000,000원
  - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
  - 3등: 5개 번호 일치 / 1,500,000원
  - 4등: 4개 번호 일치 / 50,000원
  - 5등: 3개 번호 일치 / 5,000원
- 구매한 로또와 당첨 번호를 비교해 일치 개수 및 보너스 일치 여부 산출
- 총 수익률 계산 (총 상금 / 구입 금액 × 100, 소수점 둘째 자리)

### 입력
- 구입 금액 입력
- 당첨 번호 입력 (쉼표로 구분)
- 보너스 번호 입력

### 출력
- 구매한 로또 개수 출력
- 발행된 로또 번호 목록 출력 (오름차순 정렬)
- 당첨 통계 출력 (3등과 2등의 표기 구분: "5개 일치" vs "5개 일치, 보너스 볼 일치")
- 총 수익률 출력

### 예외 처리
- 잘못된 값 입력 시 `IllegalArgumentException`을 발생시키고, `[ERROR]`로 시작하는 메시지를 출력한 뒤 해당 입력 단계부터 다시 입력받는다.
- 구입 금액
  - 숫자가 아닌 값
  - 1,000원 단위가 아닌 값
  - 0 이하의 값
- 당첨 번호
  - 6개가 아닌 경우
  - 1 ~ 45 범위를 벗어난 값
  - 중복된 번호가 있는 경우
  - 숫자가 아닌 값
- 보너스 번호
  - 1 ~ 45 범위를 벗어난 값
  - 당첨 번호와 중복되는 경우
  - 숫자가 아닌 값

## 구현 전략

### 책임 분리 (MVC)
- **Model**: 도메인 객체(`LottoNumber`, `Lotto`, `Lottos`, `LottoMachine`, `WinningLotto`, `Rank`, `LottoResult`, `Money`)
- **View**: `InputView`(입력 + 재입력 처리), `OutputView`(출력)
- **Controller**: `LottoController`가 입력 → 발행 → 비교 → 출력 흐름을 조립
- **Util**: `NumberGenerator` 인터페이스로 랜덤 생성기를 추상화 (테스트에서 고정값 주입 가능)

### 구현 순서 (TDD: 작은 단위 → 큰 단위)
1. `LottoNumber` — 1~45 범위 검증
2. `Lotto` — 6개, 중복 없음 검증
3. `Money` — 1,000원 단위 검증, 구매 가능 수량 계산
4. `LottoMachine` — 무작위 6개 번호로 로또 발행
5. `WinningLotto` — 당첨 번호 + 보너스 번호 (중복 검증)
6. `Rank` (enum) — 일치 개수/보너스 일치 여부 → 등수 매핑, 상금 보유
7. `LottoResult` — 등수별 집계 + 총 수익률 계산
8. `InputView` / `OutputView` — 재입력 루프 포함
9. `LottoController` — 전체 흐름 조립

### 핵심 설계 원칙
- 모든 도메인 객체는 **생성자에서 유효성 검증**한다.
- 랜덤 번호 생성기는 인터페이스로 분리해 **테스트 시 고정값 주입**이 가능하도록 한다.
- 재입력 처리는 **View 계층**에서 `while` 루프로 예외를 잡아 다시 입력받는다.
- 등수 판정은 `Rank` enum이 책임지며, 외부에서 `if/else`로 분기하지 않는다.
- 각 기능 구현이 끝나면 **기능 단위로 커밋**한다.
